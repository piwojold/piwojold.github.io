// Czym jest Obsidian i po co go używać do OSINT

Lokalny „mózg” analityka

Obsidian to edytor notatek Markdown działający w 100% lokalnie — żaden serwer nie widzi Twoich danych.

🔒

Lokalność danych

Wszystkie notatki to pliki .md na Twoim dysku. Możesz zaszyfrować cały folder. Żadna chmura, żaden vendor.

🕸

Graf połączeń

Automatyczna wizualizacja relacji między osobami, firmami i zdarzeniami — jak Maltego, ale dla Twoich notatek.

⚡

Szablony podmiotów

Raz zaprojektowany formularz dla osoby/firmy/domeny wypełniasz w sekundy. Obsidian sam tworzy powiązania.

🔌

Ekosystem wtyczek

Ponad 1000 bezpłatnych wtyczek community — od Kanban przez SQL-like queries po automatyczne raporty PDF.

📄

Markdown = trwałość

Format .md czyta każdy edytor tekstu. Twoje notatki będą czytelne za 20 lat, niezależnie od producenta softu.

🤖

Automatyzacja

Skrypty JS w szablonach, SQL-like queries Dataview i shell commands pozwalają generować raporty jednym kliknięciem.

// Poziom 01 — Podstawowy

Fundament vaultu OSINT

Instalacja, interfejs, struktura folderów, szablony i nawyki dokumentacyjne. Żadnej wiedzy wstępnej nie zakładamy.

01
Instalacja Obsidian i pierwsze uruchomienie
Pobierz aplikację, utwórz vault i poznaj każdy element interfejsu — od paska bocznego po edytor.
▾
⚠️
OPSEC na start: Nigdy nie synchronizuj vaultu OSINT przez iCloud, Google Drive, OneDrive ani Dropbox. Twoje źródła i metody powinny pozostać wyłącznie na Twoim urządzeniu. Jeśli potrzebujesz synchronizacji — użyj lokalnego Syncthing lub zaszyfrowanego dysku zewnętrznego.
1
Pobierz Obsidian

Wejdź na obsidian.md i kliknij przycisk Download. Strona automatycznie wykryje Twój system:

  • Windows: pobierz .exe (instalator) lub .zip (wersja przenośna — polecana do uruchamiania z zaszyfrowanego dysku)
  • macOS: pobierz .dmg, przeciągnij ikonę do Applications
  • Linux: pobierz .AppImage, nadaj prawa wykonania: chmod +x Obsidian-*.AppImage, następnie uruchom

Instalacja standardowa — Next → Install → Finish. Żadne konto nie jest wymagane do pracy lokalnej.

2
Opcjonalnie: przygotuj zaszyfrowany kontener

Zanim założysz vault, warto przygotować zaszyfrowane miejsce na dane. Opcje:

  • Windows: włącz BitLocker na dysku (Panel sterowania → Szyfrowanie dysku funkcją BitLocker) lub utwórz wolumin VeraCrypt (veracrypt.fr)
  • macOS: Disk Utility → File → New Image → Image from Folder — wybierz szyfrowanie AES-256
  • Linux: cryptsetup + LUKS lub EncFS

Możesz pominąć ten krok na początku i wrócić do niego później — Obsidian działa z dowolnym folderem.

3
Utwórz nowy vault — Twój „sejf danych”

Po pierwszym uruchomieniu Obsidian pokazuje ekran powitalny. Wybierz Create new vault.

Obsidian — Welcome
Open vault
➕ Create new vault
📁 Open folder as vault
☁️ Sign in to sync

W polu Vault name wpisz: OSINT-Vault. W polu Location kliknij Browse i wskaż folder na zaszyfrowanej partycji (lub dowolny folder). Kliknij Create.

4
Rozpoznaj interfejs — 5 stref Obsidian

Po utworzeniu vaultu zobaczysz główne okno. Składa się z 5 obszarów:

OSINT-Vault — Obsidian
📁🔍⭐🕸⚙️
PLIKI VAULTU
📄 Witaj.md
← lewa kolumna
# Witaj w Obsidian
← środek: edytor notatek
Tutaj piszesz i czytasz notatki w formacie Markdown.
WŁAŚCIWOŚCI
← prawa kolumna
  • Lewa ikony (ribbon): pionowy pasek — przełączasz widoki (pliki, wyszukiwarka, zakładki, graf), na dole ikona ⚙️ to Settings
  • Lewa kolumna (sidebar): eksplorator plików — widzisz tu foldery i pliki swojego vaultu jak w eksploratorze systemu
  • Środek (editor): tu piszesz i czytasz notatki w formacie Markdown — to główna przestrzeń robocza
  • Prawa kolumna: właściwości notatki (frontmatter YAML), linki przychodzące, lokalny graf
  • Pasek kart (u góry): otwarte notatki jak karty w przeglądarce — możesz mieć kilka otwartych jednocześnie
5
Wyłącz telemetrię i ustaw ciemny motyw

Naciśnij Ctrl+, (przecinek) lub kliknij ikonę ⚙️ w lewym dolnym rogu — otworzy się okno Settings. Przejdź do sekcji General:

  • Odznacz opcję Send usage statistics to Obsidian

Kliknij Appearance w lewym menu Settings:

  • Base color scheme → wybierz Dark
  • Kliknij Manage przy Themes, wyszukaj Minimal i zainstaluj — czyste, profesjonalne renderowanie do eksportu PDF

Zamknij Settings klikając Esc.

6
Odblokuj wtyczki społecznościowe

Domyślnie Obsidian pozwala używać tylko własnych wtyczek. Odblokuj Community plugins:

Otwórz Ctrl+, → kliknij Community plugins w lewym menu → kliknij Turn on community plugins → pojawi się ostrzeżenie → kliknij Turn on.

ℹ️
Obsidian pyta o zgodę, bo wtyczki community mają dostęp do plików. Instaluj tylko wtyczki z wysoką liczbą pobrań i aktywnymi aktualizacjami — to sygnał, że są sprawdzone przez społeczność.
7
Zainstaluj 4 kluczowe wtyczki startowe

W sekcji Community plugins kliknij Browse. Dla każdej wtyczki: wpisz nazwę, kliknij wynik, kliknij Install, poczekaj, kliknij Enable:

TemplaterDataviewTag WranglerCalendar
  • Templater — inteligentne szablony z automatycznie wypełnianymi polami (data, tytuł, ID sprawy)
  • Dataview — odpytuje notatki jak bazę danych SQL: „pokaż mi wszystkie aktywne podmioty z ostatnich 7 dni”
  • Tag Wrangler — masowe zarządzanie tagami: przemianuj, połącz duplikaty, ukryj rzadkie
  • Calendar — widok kalendarza z dzięnnymi notatkami wywiadowczymi

Po instalacji wszystkich czterech zamknij okno wtyczek (Esc).

✅
Efekt po tej lekcji: Masz zainstalowany Obsidian z odblokowanymi wtyczkami, ciemnym motywem i wyłączoną telemetrią. Wiesz jak poruszać się po interfejsie.
02
Markdown w 10 minut — język notatek OSINT
Składnia Markdown, wikilinki, tagi, frontmatter YAML — cztery rzeczy których używasz codziennie.
▾
📚
Czym jest Markdown? To prosty sposób formatowania tekstu za pomocą znaków takich jak #, * czy -. Piszesz zwykły tekst, a Obsidian renderuje go jako sformatowany dokument. Pliki .md otworzysz w każdym edytorze — nawet w Notatniku.
1
Utwórz pierwszą notatkę i włącz Live Preview

Naciśnij Ctrl+N — w środkowym panelu otworzy się pusta notatka. Tytuł „Untitled” jest edytowalny — wpisz: Jan Kowalski — Test i naciśnij Enter.

Obsidian domyślnie działa w trybie Live Preview — formatowanie pojawia się od razu podczas pisania. Możesz przełączyć na tryb Source (surowy Markdown) ikoną ✎ w prawym górnym rogu edytora lub na Reading View (czysty podgląd) ikoną 👀.

2
Podstawowa składnia — przepisz ten przykład

Wpisz poniższy tekst w otwartej notatce i obserwuj jak Obsidian formatuje go na żywo:

# Jan Kowalski ← nagłówek H1 (tytuł sekcji) ## Identyfikatory ← nagłówek H2 (podsekcja) ### Notatki analityczne ← nagłówek H3 **Imię i nazwisko:** Jan Kowalski ← pogrubienie *Pseudonim:* „Kowal” ← kursywa - Email: jan@example.com ← lista punktowana - Telefon: nieznany - LinkedIn: linkedin.com/in/jkowal 1. Sprawdź WHOIS domeny ← lista numerowana 2. Przeszukaj social media 3. Zweryfikuj adres > WAŻNE: podmiot aktywny od 2022 ← cytat/notatka wyróżniona | Platforma | Username | Status | ← tabela |-----------|------------|---------| | Twitter | @jkowal | aktywny | | GitHub | jkowal-dev | aktywny | - [x] Zidentyfikowany ← checkbox zaznaczony - [ ] Zweryfikowany ← checkbox do zrobienia

Każda z tych konstrukcji będzie Ci potrzebna w notatkach OSINT. Ćwicz je aż piszesz z pamięci.

3
Wikilinki — serce Obsidian

Wikilink to podwójny nawias kwadratowy [[nazwa notatki]]. To najważniejsza funkcja Obsidian — tworzy aktywne połączenia między notatkami widoczne w grafie:

Pracodawca: [[Firma XYZ Sp. z o.o.]] Powiązana osoba: [[Anna Nowak]] Zdarzenie: [[Atak phishingowy 2024-03-15]] Z aliasem: [[Anna Nowak|A. Nowak]] ← tekst po | to wyświetlana nazwa

Jeśli notatka o „Firma XYZ” jeszcze nie istnieje — link wyświetli się szarą czcionką (unresolved). Kliknij go i Obsidian automatycznie otworzy nową notatkę z tą nazwą.

💡
Zacznij wpisywać [[ i Obsidian pokaże podpowiedzi ze wszystkich notatek w vaulcie — dokładnie jak autocomplete. Wpisz fragment nazwy i wybierz strzałkami.
4
Tagi — etykiety do szybkiego filtrowania

Tag to słowo poprzedzone hashem: #podmiot/osoba. Obsidian traktuje / jako separator hierarchii:

#podmiot/osoba ← typ wpisu #status/aktywny ← stan dochodzenia #priorytet/wysoki ← pilność #zrodlo/linkedin ← skąd pochodzi informacja

Tagi wpisz bezpośrednio w treści notatki lub w frontmatter (pole tagi:). W lewym panelu — ikona znaczka — zobaczysz wszystkie tagi z liczbą notatek.

5
Frontmatter YAML — metadane notatki jako baza danych

Frontmatter to blok na samej górze notatki, otoczony ---. Obsidian traktuje go jak pola bazy danych — możesz je później odpytywać przez Dataview:

--- typ: osoba status: aktywny narodowosc: PL data_utworzenia: 2024-11-20 tagi: [podmiot/osoba, priorytet/wysoki] --- # Jan Kowalski Tutaj zaczyna się właściwa treść notatki.

Ważna zasada: klucze YAML pisz bez polskich znaków i spacji — używaj podkreślników: data_urodzenia, nie data urodzenia. Wielkie litery w kluczach mogą sprawiać problemy z Dataview — pisz małymi.

✅
Efekt: Znasz składnię Markdown, wikilinki, tagi i frontmatter. To 95% tego, czego używasz na co dzień w Obsidian.
03
Struktura folderów — architektura wywiadowcza
Zaprojektuj hierarchię katalogów odzwierciedlającą realne procesy śledcze — tak żebyś za rok wiedział gdzie co leży.
▾
📁
Dlaczego struktura jest ważna? Chaotyczna struktura oznacza, że za 3 miesiące nie znajdziesz niczego. Zainwestuj 15 minut teraz — zaoszczędzisz godziny w przyszłości.
1
Jak tworzyć foldery w Obsidian

W lewym panelu (eksplorator plików) kliknij prawym przyciskiem myszy na pustym obszarze → wybierz New folder. Możesz też kliknąć ikonę folderu w górnym pasku eksploratora.

Trick: Obsidian sortuje foldery alfabetycznie. Numerując je (00, 01, 02…) wymuszasz stałą kolejność niezależnie od nazwy. Podfoldery tworzysz klikając prawym na folder nadrzędny → New folder.

2
Utwórz kompletną strukturę vaultu
OSINT-Vault/ ├── 00_META/ ← szablony, skrypty, taksonomia tagów │ ├── Templates/ │ └── Scripts/ ├── 01_DOCHODZENIA/ ← aktywne sprawy │ └── CASE_001_PRZYKŁAD/ ├── 02_PODMIOTY/ ← baza podmiotów │ ├── Osoby/ │ ├── Firmy/ │ ├── Domeny/ │ └── Konta-SM/ ├── 03_ZDARZENIA/ ← incydenty z datami (oś czasu) ├── 04_NARZEDZIA/ ← notatki o narzędziach OSINT ├── 05_RAPORTY/ ← gotowe raporty końcowe └── 06_ARCHIWUM/ ← zamknięte sprawy w .zip
3
Skonfiguruj domyślny folder i folder załączników

Otwórz Ctrl+, → Files & Links:

  • Default location for new notes → In the folder specified below → wpisz: 01_DOCHODZENIA
  • Default location for new attachments → In subfolder under current folder → wpisz: _attachments

Dzięki temu każda nowa notatka startuje we właściwym miejscu, a wklejone screenshoty trafiają do _attachments w folderze aktywnego case’u.

4
Stwórz plik INDEX — punkt wejścia dochodzenia

Kliknij folder CASE_001_PRZYKŁAD prawym → New note → nazwij ją INDEX. Wpisz:

--- typ: dochodzenie-index case_id: CASE-001 cel: "Opłacono Twoim imieniu" status: aktywny data_otwarcia: 2024-11-20 priorytet: wysoki --- # CASE-001 — [Nazwa Sprawy] ## Kluczowe podmioty - [[Osoba 1]] - [[Firma ABC]] ## Oś czasu - 2024-11-20: Otwarcie sprawy ## Status zadań - [ ] Zebranie danych OSINT - [ ] Weryfikacja źródeł - [ ] Sporządzenie raportu
✅
Efekt: Vault ma profesjonalną strukturę. Każde nowe dochodzenie zaczynasz od stworzenia folderu i pliku INDEX z checklistą zadań.
04
Szablony podmiotów — formularze wywiadowcze
Zbuduj gotowe formularze dla osób, firm i domen. Każdy nowy podmiot wypełnisz w 30 sekund.
▾
1
Skonfiguruj Templater — wskaż folder szablonów

Otwórz Ctrl+, → kliknij Templater → w polu Template folder location wpisz: 00_META/Templates. Włącz Trigger Templater on new file creation.

2
Utwórz szablon profilu osoby

Kliknij folder 00_META/Templates prawym → New note → nazwij: TPL_Osoba. Wklej:

--- typ: osoba imie_nazwisko: "<% tp.file.title %>" pseudonimy: [] data_urodzenia: "" narodowosc: "" lokalizacja: "" status: aktywny wiarygodnosc: C powiazane_firmy: [] data_utworzenia: <% tp.date.now("YYYY-MM-DD") %> tagi: [podmiot/osoba, status/do-weryfikacji] --- # <% tp.file.title %> ## Identyfikatory kontaktowe | Typ | Wartość | Status | |---------|---------|----------| | Email | | nieznany | | Telefon | | nieznany | | Adres | | nieznany | ## Profile online | Platforma | URL / Username | Aktywny? | |-----------|----------------|----------| | LinkedIn | | | | Twitter/X | | | | GitHub | | | ## Powiązania - Pracodawca: [[]] - Współpracownicy: [[]] - Powiązane zdarzenia: [[]] ## Śródła informacji | Data | Śródło | Informacja | Wiarygodność | |------|-------|------------|-------------| ## Notatki analityczne > Wpisuj tu obserwacje, hipotezy i wnioski. ## Do weryfikacji - [ ] Potwierdź tożsamość - [ ] Sprawdź social media - [ ] Zweryfikuj powiązania

Znacznik <% tp.file.title %> zostaje zastąpiony nazwą pliku — notatka „Jan Kowalski” automatycznie wypelni pole imie_nazwisko i nagłówek #.

3
Jak użyć szablonu w praktyce

Sposób 1 — ręczny: Otwórz nową notatkę (Ctrl+N), nadaj jej nazwę (np. „Jan Kowalski”), naciśnij Ctrl+P → wpisz Templater: Insert Template → wybierz TPL_Osoba.

Sposób 2 — automatyczny (polecany): W ustawieniach Templater → Folder Templates → kliknij + → ustaw: Folder = 02_PODMIOTY/Osoby, Template = TPL_Osoba. Teraz każda notatka utworzona w tym folderze automatycznie dostaje szablon osoby.

4
Skala wiarygodności — klasyfikuj źródła jak wywiad

Pole wiarygodnosc: w szablonie używa skali wzorowanej na standardach wywiadowczych:

A — Potwierdzone: min. 2 niezależne źródła B — Prawdopodobnie prawdziwe: jedno wiarygodne źródło C — Możliwe: nieweryfikowalne, ale logicznie spójne D — Wątpliwe: sprzeczne z innymi informacjami F — Fałszywe: zweryfikowane jako nieprawdziwe

W raporcie końcowym cytuj wyłącznie źródła A i B. Informacje klasy C to wskazówki do dalszych poszukiwań, nigdy fakty.

✅
Efekt: Każdy nowy podmiot dokumentujesz w 30 sekund. Dane trafiają do frontmatter — Dataview będzie mógł je później odpytywać automatycznie.
05
Tagowanie i wyszukiwanie — znajdź cokolwiek w sekundy
Zbuduj taksonomie tagów OSINT i opanuj zaawansowane wyszukiwanie pełnotekstowe z operatorami.
▾
1
Zaprojektuj hierarchię tagów — zapisz ją raz na zawsze

Utwórz notatkę 00_META/Taksonomia-Tagów.md i skopiuj poniższą strukturę. Wróć do niej za każdym razem gdy chcesz dodać nowy tag:

## Typ podmiotu #podmiot/osoba ← profil fizycznej osoby #podmiot/firma ← spółka, organizacja #podmiot/domena ← domain name, IP, infrastruktura #podmiot/konto-sm ← konto social media ## Status #status/aktywny #status/do-weryfikacji #status/zarchiwizowany ## źródło informacji #zrodlo/socialmedia #zrodlo/whois #zrodlo/dns-pasywny #zrodlo/rejestry-publiczne #zrodlo/web-clipper #zrodlo/shodan ## Priorytet #priorytet/wysoki #priorytet/sredni #priorytet/niski
2
Zaawansowane wyszukiwanie — operatory

Naciśnij Ctrl+Shift+F aby otworzyć wyszukiwarkę globalną. Obsidian obsługuje operatory:

# Wszystkie aktywne osoby tag:#podmiot/osoba tag:#status/aktywny # W konkretnym folderze path:01_DOCHODZENIA/CASE_001 # Wyklucz zarchiwizowane tag:#podmiot -tag:#status/zarchiwizowany # Linia zawierająca słowo kluczowe line:(email OR telefon) # Po dacie modyfikacji mtime:2024-11-13..2024-11-20
3
Zapisz wyszukiwania jako zakładki i Quick Switcher

Po wpisaniu zapytania kliknij ikonę zakładki (🔖) w prawym górnym rogu panelu wyszukiwania → Save search. Zakładki znajdziesz klikając ikonę ⭐ w lewym pasku — jeden klik uruchamia aktualny filtr.

Quick Switcher: naciśnij Ctrl+O — zacznij wpisywać fragment nazwy notatki i przełączaj się między plikami bez myszy. To najszybszy sposób nawigacji w dużym vaulcie — ćwicz od pierwszego dnia.

🎯
Efekt po poziomie podstawowym: Masz w pełni funkcjonalny vault — struktura, szablony, tagowanie, wyszukiwanie. Czas na poziom średni: graf powiązań i automatyczne raporty Dataview.

Graf relacji, Dataview i web clipping

Wizualizuj sieć powiãzañ, odpytuj vault jak bazę danych i automatycznie importuj strony do notatek.

06
Graph View — mapa relacji wywiadowczych
Skonfiguruj graf, koloruj węzły według typów podmiotów i technika pivot przez sieć.
▾
🕸
Czym jest Graph View? Wizualizacja wszystkich wikilinków jako węzłów (notatki) i krawędzi (połączenia). Im więcej linków ma notatka, tym większy węzeł. Podmiot powiązany z wieloma osobami będzie centralnym hubem.
1
Global Graph vs Local Graph
  • Global Graph (Ctrl+G) — cały vault. Do widoku z lotu ptaka.
  • Local Graph (ikona grafu w górnym rogu) — tylko notatki połączone z aktualnie otwartą. Główne narzędzie analityczne — odpowiednik "Expand" w Maltego.
2
Konfiguracja filterów i Display

Panel po prawej stronie grafu — sekcja Filters:

  • Search: tag:#podmiot — pokazuj tylko podmioty
  • Włącz Tags — węzły tagów widoczne osobno
  • Wyłącz Orphans — ukryj notatki bez połączeń

Sekcja Display: Node size 8, Link thickness 2, włącz Arrow.

3
Kolorowanie węzłów według typów podmiotów

Panel grafu → sekcja Groups → New group:

tag:#podmiot/osoba → kolor: #00d4ff (niebieski) tag:#podmiot/firma → kolor: #a78bfa (fioletowy) tag:#podmiot/domena → kolor: #f97316 (pomarańczowy) tag:#priorytet/wysoki → kolor: #ef4444 (czerwony)

Graf staje się czytelna mapą kolorystyczną — natychmiast widoczne typy podmiotów i gęstość powiązań.

4
Technika pivot — nawigacja przez sieć

Otwórz Local Graph przy notatce osoby. Ustaw Depth na 2. Kliknij dwa razy na węzeł firmy — Obsidian otworzy jej notatkę i Graf odświeży się względem niej. Przeskakujesz po sieci węzeł po węźle jak w Maltego.

💡
Ustaw Local Graph w osobnym panelu (przeciągnij kartę na bok) — edytor po lewej, graf po prawej. Edytujesz notatki i na żywo widzisz zmiany w sieci.
07
Dataview — SQL dla Twoich notatek
Zapytania automatycznie agregujące dane z dziesiątek notatek. Buduj żywe dashboardy wywiadowcze.
▾
📊
Czym jest Dataview? Wtyczka traktująca vault jak bazę danych. Każde pole frontmatter staje się kolumną. Pytasz: "Pokaż mi wszystkie aktywne osoby posortowane od najnowszej".
1
Pierwsza tabela — aktywne podmioty

Utwórz notatkę 00_META/Dashboard.md i skopiuj:

Dwa tryby tagów Templater

// Generuje unikalny ID: CASE-202411-1234 function caseId() { const now = new Date(); const y = now.getFullYear(); const m = String(now.getMonth() + 1).padStart(2, '0'); const r = Math.floor(Math.random() * 9000) + 1000; return `CASE-${y}${m}-${r}`; } module.exports = caseId;

Wywołanie w szablonie: <% tp.user.caseId() %>

DataviewJS — zaawansowane agregacje wywiadowcze
JavaScript API Dataview: alerty o przestarzałych danych, macierze powiązań i heatmapa aktywności.
▾
1
DQL vs DataviewJS — kiedy używać czego

Zwykły Dataview (DQL) wystarczy do ~90% zadań. DataviewJS (blok ```dataviewjs) używaj gdy:

  • Potrzebujesz obliczeń (np. ile dni minęło od aktualizacji)
  • Chcesz warunkowego formatowania (podkreśl czerwono jeśli...)
  • Budujesz zagnieżdżone agregacje niemożliwe w DQL
2
Alert — podmioty bez aktualizacji >7 dni
```dataviewjs const DAYS = 7; const stale = dv.pages('#podmiot AND #status/aktywny') .where(p => (Date.now() - p.file.mtime) / 86400000 > DAYS); if (stale.length === 0) { dv.paragraph("✅ Wszystkie aktywne podmioty aktualne."); } else { dv.header(4, `⚠️ ${stale.length} podmiotów bez aktualizacji >${DAYS} dni:`); dv.table(["Podmiot", "Ostatnia zmiana"], stale.map(p => [p.file.link, p.file.mtime.toFormat("dd.MM.yyyy")])); } ```
3
Macierz powiązań osoba → firma
```dataviewjs const osoby = dv.pages('"02_PODMIOTY/Osoby"') .where(p => p.powiazane_firmy?.length > 0); dv.table(["Osoba", "Powiązane firmy", "Status"], osoby.map(p => [ p.file.link, dv.array(p.powiazane_firmy).join(", "), p.status ]) ); ```
4
Heatmapa aktywności zbierania danych
```dataviewjs const byDay = {}; dv.pages('#zrodlo').forEach(p => { const d = p.data_archiwizacji?.toString().slice(0,10); if(d) byDay[d] = (byDay[d]||0)+1; }); const sorted = Object.entries(byDay).sort().slice(-30); dv.table(["Data", "Nowe źródła", "Wykres"], sorted.map(([d,n]) => [d, n, "█".repeat(Math.min(n,20))])); ```

Skrypt Python — Shodan → notatka domeny
import shodan, os, sys from datetime import datetime api = shodan.Shodan(os.environ['SHODAN_KEY']) ip, vault = sys.argv[1], sys.argv[2] host = api.host(ip) ports = [ f"- **{s['port']}/{s['transport']}** — {s.get('product','?')}" for s in host['data'] ] lines = [ "---", f"typ: infrastruktura", f"ip: \"{host['ip_str']}\"", f"org: \"{host.get('org','?')}\"", f"kraj: \"{host.get('country_name','?')}\"", f"ostatni_skan: \"{datetime.now().strftime('%Y-%m-%d')}\"", "tagi: [podmiot/ip, zrodlo/shodan, status/do-weryfikacji]", "---", f"# {host['ip_str']}", "", "## Otwarte porty", ] + ports + ["", "## Notatki analityczne"] path = f"{vault}/02_PODMIOTY/Domeny/{ip}.md" open(path, 'w', encoding='utf-8').write('\n'.join(lines))

Shell Command w Obsidian: python ~/scripts/shodan.py {{prompt:IP}} {{vault_path}}

import csv, os TYPE_MAP = { 'Person': ('02_PODMIOTY/Osoby', 'podmiot/osoba'), 'Domain': ('02_PODMIOTY/Domeny', 'podmiot/domena'), 'Company': ('02_PODMIOTY/Firmy', 'podmiot/firma'), } VAULT = "/sciezka/OSINT-Vault" with open('maltego_export.csv', encoding='utf-8') as f: for row in csv.DictReader(f): folder, tag = TYPE_MAP.get(row['Type'], ('02_PODMIOTY', 'podmiot/inne')) val = row['Value'] lines = [ "---", f"typ: {row['Type'].lower()}", f"wartosc: \"{val}\"", f"tagi: [{tag}, zrodlo/maltego, status/do-weryfikacji]", "---", f"# {val}", "", "> Importowane z Maltego. Wymaga weryfikacji." ] safe = val.replace('/','-')[:80] open(f"{VAULT}/{folder}/{safe}.md",'w').write('\n'.join(lines))
