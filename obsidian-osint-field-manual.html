Obsidian to lokalny edytor notatek oparty na plikach Markdown. Dla pracy OSINT jest idealny — dane nigdy nie opuszczają Twojego komputera, możesz budować złożone bazy wiedzy i wizualizować powiązania między podmiotami. Brak konta, brak chmury, pełna kontrola.
1
Pobierz Obsidian

Wejdź na obsidian.md i pobierz wersję dla swojego systemu (Windows, macOS, Linux). Obsidian jest bezpłatny do użytku osobistego i komercyjnego. Instalacja nie wymaga uprawnień administratora — dostępna też wersja portable.

2
Utwórz vault OSINT

Vault to zwykły folder na dysku. Utwórz go w bezpiecznym miejscu — najlepiej na zaszyfrowanym wolumenie (VeraCrypt / BitLocker). Nazwij np. OSINT_Vault_2025. Jeden vault = jeden projekt lub tematyczny obszar pracy analitycznej.

3
Poznaj interfejs

Lewy panel: nawigator plików i wyszukiwarka. Środek: edytor z Live Preview. Prawy panel: linki zwrotne i właściwości pliku. Pasek boczny przełącza Ctrl+B. Górny pasek: nazwa pliku, gwiazdka = niezapisany.

4
Konfiguracja wstępna bezpieczeństwa

Ustawienia → O programie → wyłącz 'Send crash reports' i 'Send usage data'. Ustawienia → Pliki i linki → wyłącz 'Detect all file extensions'. Vault przechowuj wyłącznie na lokalnym, zaszyfrowanym dysku.

4
Callouts — semantyczne oznaczanie danych

Callouts (> [!TYP]) to bloki wizualne oznaczające typ informacji. Kluczowe dla OSINT: [!INFO] fakt potwierdzony, [!WARNING] niezweryfikowane, [!DANGER] wrażliwe PII, [!NOTE] hipoteza robocza.

5
Formatowanie inline

**bold** dla kluczowych danych, ==highlight== dla oznaczeń (wymaga pluginu), `kod` dla identyfikatorów (IP, hashi, ID kont). Użyj formatowania oszczędnie — nadmiar niszczy czytelność operacyjną.

Utrata vaulta OSINT to utrata miesięcy pracy. Strategia backupu musi być wbudowana w rutynę — nie można o niej pamiętać ad hoc. Poznaj opcje: lokalne, sieciowe i E2EE, z uwzględnieniem wymagań bezpieczeństwa.
1
Backup lokalny — minimum

Regularny backup vault na zewnętrzny dysk zaszyfrowany. Strategia 3-2-1: 3 kopie, na 2 różnych nośnikach, 1 poza lokalizacją. Automatyzuj przez skrypt lub Time Machine / Windows Backup.

2
Git jako backup z historią

Vault to folder z plikami tekstowymi — idealny dla Git. git init w folderze vault. Plugin 'Obsidian Git' commituje zmiany automatycznie co N minut. Historia zmian każdej notatki przez git log.

3
Syncthing — P2P bez chmury

Syncthing synchronizuje między własnymi urządzeniami bez zewnętrznych serwerów. Szyfrowanie TLS, zero danych na obcych serwerach. Idealne jeśli pracujesz na laptopie i desktopie równolegle.

4
Obsidian Sync (E2EE)

Oficjalna synchronizacja Obsidiana z szyfrowaniem end-to-end — klucz szyfrowania tylko u Ciebie. 4 USD/miesiąc. Historia wersji 12 miesięcy. Nie polecane dla materiałów objętych tajemnicą zawodową.

Skrypt automatycznego backupu (Linux/macOS) 
#!/bin/bash
# backup-vault.sh — uruchamiaj przez cron co 6 godzin

VAULT_PATH="$HOME/OSINT_Vault"
BACKUP_DIR="/mnt/encrypted-backup/obsidian-backups"
DATE=$(date +%Y%m%d_%H%M)

# Kompresja i szyfrowanie GPG
tar -czf - "$VAULT_PATH" | \
  gpg --symmetric --cipher-algo AES256 \
      --output "$BACKUP_DIR/vault_$DATE.tar.gz.gpg"

# Zachowaj tylko 30 ostatnich backupów
ls -t "$BACKUP_DIR"/vault_*.gpg | tail -n +31 | xargs rm -f

echo "Backup ukończony: vault_$DATE.tar.gz.gpg"

# Crontab: co 6 godzin
# 0 */6 * * * /home/user/backup-vault.sh >> /var/log/vault-backup.log 2>&1

```dataviewjs
const pages = dv.pages('"01_Podmioty"');

// Statystyki zbiorcze
const stats = {
  total: pages.length,
  osoby: pages.where(p => p.type === "osoba").length,
  firmy: pages.where(p => p.type === "firma").length,
  aktywne: pages.where(p => p.status === "aktywny").length,
  potwierdzone: pages.where(p => p.weryfikacja === "potwierdzony").length,
  krytyczne: pages.where(p => p.priorytet === 1).length,
};

// Statystyki procentowe
const pctPotw = Math.round(stats.potwierdzone / stats.total * 100) || 0;

dv.header(3, "Podsumowanie śledztwa");
dv.table(
  ["Metryka", "Wartość", "Info"],
  [
    ["Łącznie podmiotów", stats.total, ""],
    ["Osoby", stats.osoby, "${Math.round(stats.osoby/stats.total*100)}%"],
    ["Firmy", stats.firmy, "${Math.round(stats.firmy/stats.total*100)}%"],
    ["Aktywnych", stats.aktywne, ""],
    ["Potwierdzonych", stats.potwierdzone, "${pctPotw}% pokrycia"],
    ["Krytycznych", stats.krytyczne, "🔴 priorytet 1"],
  ]
);
```

#!/usr/bin/env python3
"""flatten-obsidian.py — spłaszcza ![[osadzenia]] do eksportu Pandoc"""

import re
import sys
from pathlib import Path

VAULT_PATH = Path("/home/user/OSINT_Vault")

def flatten(file_path: Path, depth=0, visited=None) -> str:
    if visited is None:
        visited = set()
    if file_path in visited or depth > 10:
        return f"[CIRCULAR REF: {file_path.name}]"
    visited.add(file_path)

    try:
        content = file_path.read_text(encoding="utf-8")
    except FileNotFoundError:
        return f"[MISSING: {file_path.name}]"

    # Zamień ![[plik#sekcja]] na zawartość
    def replace_embed(match):
        embed_name = match.group(1).split("#")[0].strip()
        candidates = list(VAULT_PATH.rglob(f"{embed_name}.md"))
        if not candidates:
            return f"[NOT FOUND: {embed_name}]"
        return flatten(candidates[0], depth + 1, visited)

    return re.sub(r"![[([^]]+)]]", replace_embed, content)

if __name__ == "__main__":
    input_file = Path(sys.argv[1])
    print(flatten(input_file))

OSINT w zespole wymaga koordynacji: kto pracuje nad czym, jak unikać nadpisywania danych, jak scalać ustalenia z różnych analityków. Obsidian nie ma natywnej funkcji współpracy — ale są sprawdzone strategie.
1
Git jako platforma współpracy

Wspólne repozytorium Git (prywatne, zaszyfrowane git-crypt). Każdy analityk pracuje na własnej gałęzi. Pull request = review ustaleń przed merge do gałęzi głównej. Historia commitów = audit trail kto co dodał.

2
Podział odpowiedzialności (Vault splits)

Każdy analityk ma własną gałąź folderów: 01_Podmioty/JK/ (analityk 1), 01_Podmioty/AK/ (analityk 2). Merge przez Git. Wspólne pliki: Dashboard.md, taksonomia tagów, szablony — tylko przez PR.

3
Standard notatek dla teamów

Pole 'operator' w frontmatter identyfikuje autora. Sekcja 'Notatki operatora' na końcu każdego pliku z podpisem i datą. Nigdy nie edytuj cudzej sekcji bez merge — zostawiaj komentarz jako nową notatkę.

4
Review i walidacja krzyżowa

Dwóch analityków niezależnie weryfikuje kluczowe ustalenia. Plugin 'Review' dodaje adnotacje do notatek. Dataview: WHERE weryfikacja != 'potwierdzony' AND operator != 'twoj-nick' — lista do Twojego review.

Workflow Git dla zespołu OSINT 
# Workflow Git — team OSINT

## Setup repozytorium (jednorazowo)
git init OSINT_Vault
cd OSINT_Vault
git-crypt init                        # szyfrowanie wrażliwych plików
git-crypt add-gpg-user <GPG_KEY_ID>   # dodaj klucze GPG analityków

# .gitattributes — co szyfrować
*.md filter=git-crypt diff=git-crypt
_załączniki/** filter=git-crypt diff=git-crypt

## Codzienny workflow analityka
git checkout -b analityk-jan          # własna gałąź
# ... praca: tworzenie i edycja notatek ...
git add -p                            # selektywne dodawanie (nigdy -A)
git commit -m "osoba: Jan Kowalski — dane KRS, LinkedIn"
git push origin analityk-jan

## Merge do main (po review)
git checkout main
git pull origin main
git merge --no-ff analityk-jan
git push origin main

## Rozwiązywanie konfliktów w .md

# Użyj: git mergetool lub ręcznie przejrzyj diff
# Zasada: merge content (nie overwrite)
